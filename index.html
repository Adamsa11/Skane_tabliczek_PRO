Konfiguracja tabeli w Supabase & RLS

Instrukcja

Jeśli pomyślnie zapisujesz operacje, ale nie widzisz ich w tabeli (lub lista jest pusta), najczęstszą przyczyną jest włączone zabezpieczenie Row Level Security (RLS) w Supabase bez reguł odczytu publicznego dla klucza anon.

Wykonaj poniższe polecenie SQL w panelu **Supabase -> SQL Editor**, aby utworzyć tabelę i prawidłowo skonfigurować uprawnienia: 

-- KROK 1: Utwórz tabele jeśli jeszcze nie istnieją
CREATE TABLE IF NOT EXISTS operacje (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL,
    wozek_id bigint,
    nrkatalogowy text,
    model text,
    klient text,
    temat text,
    parametry jsonb,
    image_data text
);

CREATE TABLE IF NOT EXISTS wozki (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL,
    "NRKATALOGOWY" text UNIQUE,
    "TYP_WOZKA" text,
    "NR_FABRYCZNY" text,
    "ROK_PRODUKCJI" text,
    "UDZWIG" text,
    "SRODEK_CIEZKOSCI" text,
    "MASA_BEZ_AKUMULATORA" text,
    "MASA_AKUMULATORA_MIN" text,
    "MASA_AKUMULATORA_MAX" text,
    "NAPIECIE" text,
    "BATERIA" text,
    "TYP" text
);

-- OPCJA A (Najprostsza, wyłączenie RLS): Wykonaj te linie, aby całkowicie wyłączyć ograniczenia RLS
ALTER TABLE IF EXISTS operacje DISABLE ROW LEVEL SECURITY;
ALTER TABLE IF EXISTS wozki DISABLE ROW LEVEL SECURITY;

-- LUB OPCJA B (Bezpieczna, włączenie RLS z jawnym dostępem anonimowym):
-- ALTER TABLE IF EXISTS operacje ENABLE ROW LEVEL SECURITY;
-- ALTER TABLE IF EXISTS wozki ENABLE ROW LEVEL SECURITY;
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na odczyt operacji" ON operacje;
-- CREATE POLICY "Zezwalaj wszystkim na odczyt operacji" ON operacje FOR SELECT TO anon USING (true);
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na zapis operacji" ON operacje;
-- CREATE POLICY "Zezwalaj wszystkim na zapis operacji" ON operacje FOR INSERT TO anon WITH CHECK (true);
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na modyfikacje operacji" ON operacje;
-- CREATE POLICY "Zezwalaj wszystkim na modyfikacje operacji" ON operacje FOR UPDATE TO anon USING (true) WITH CHECK (true);
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na usuniecie operacji" ON operacje;
-- CREATE POLICY "Zezwalaj wszystkim na usuniecie operacji" ON operacje FOR DELETE TO anon USING (true);
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na odczyt wozkow" ON wozki;
-- CREATE POLICY "Zezwalaj wszystkim na odczyt wozkow" ON wozki FOR SELECT TO anon USING (true);
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na zapis wozkow" ON wozki;
-- CREATE POLICY "Zezwalaj wszystkim na zapis wozkow" ON wozki FOR INSERT TO anon WITH CHECK (true);
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na modyfikacje wozkow" ON wozki;
-- CREATE POLICY "Zezwalaj wszystkim na modyfikacje wozkow" ON wozki FOR UPDATE TO anon USING (true) WITH CHECK (true);
-- DROP POLICY IF EXISTS "Zezwalaj wszystkim na usuniecie wozkow" ON wozki;
-- CREATE POLICY "Zezwalaj wszystkim na usuniecie wozkow" ON wozki FOR DELETE TO anon USING (true);

Wyłączenie RLS (DISABLE ROW LEVEL SECURITY) gwarantuje poprawne wyświetlanie i dodawanie wpisów.
